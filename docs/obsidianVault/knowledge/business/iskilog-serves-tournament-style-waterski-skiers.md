# iSkiLog serves tournament-style waterski skiers

## Who it's for
Individual skiers practicing **tournament-style waterski** — the three classic events plus catch-all:

- **Slalom** — buoys, rope length, speed, passes
- **Tricks** — duration, hands/toes/mixed trick type
- **Jump** — jump or cuts submode; attempts/passed/made/distance
- **Other** — freeform activity + duration

This is a **single-user self-tracking** tool, not a coach/team/social platform.

## The job it does
Skiers log each practice **pass** (a [[the-app-is-a-react19-supabase-capacitor-training-log|set]]), reflect via [[notes-are-stored-as-six-structured-sections|structured notes]], organize by [[seasons-are-calendar-year-only|calendar-year season]], review [[analytics-are-computed-client-side|trends and insights]], track a [[the-app-is-a-react19-supabase-capacitor-training-log|trick library]], and export summaries.

## Why the domain shapes the product
The data model and UI are built around **real ski-session logging** — event-specific fields, tournament rope/speed conventions ([[recharts-and-jspdf-power-charts-and-exports|skiFormat]]) — not generic notes. That domain fit is the core value. See [[the-product-is-structured-self-tracking-not-social-fitness]].
